## 一、项目概览

RoadMC 是一个**物理仿真驱动、数学约束增强**的路面点云病害检测系统，严格遵循 **JTG 5210—2018《公路技术状况评定标准》**，共 38 个标签 (0-37)。

### 核心能力

1. **合成数据生成**: ISO 8608 PSD 路面谱合成 → 11 种病害物理仿真 → LiDAR 噪声模拟 → 体素下采样
2. **深度网络模型**: Swin3D 4-stage Transformer + mHC 流形约束超连接 + 可变形窗口注意力
3. **训练评估**: PyTorch Lightning 封装, FocalLoss+DiceLoss+EdgeLoss, macro mIoU

| 模块 | 文件 | 行数 | 自检 |

| Lightning 封装 | `models/model_pl.py` | ~510 | ✅ loss=4.97, 全部通过 |

**架构设计**:
```python
# 每个 ShiftedWindowTransformerBlock 的规范结构:
x_ln = LN(x)
x_attn = WindowAttention(x_ln) + x          # pre-LN residual
x_ln2 = LN(x_attn)
x_ffn = FFN(x_ln2) + x_attn                # pre-LN residual
x_out = MHCConnection(x_ffn, x_attn)       # mHC 流形约束超连接
```

**Swin3D 骨干网络**:
- 4 个阶段，通道数 `[96, 192, 384, 768]`
- 深度配置: `(2, 2, 6, 2)` (Swin-Tiny)
- 多头: `(3, 6, 12, 24)`
- FCN 分割头: 跳跃连接 + 1×1 卷积 → 38 类

**损失函数**:
$$\mathcal{L} = \lambda_1 \cdot \text{FocalLoss}(\gamma=2, \alpha=\text{class\_weights}) + \lambda_2 \cdot \text{DiceLoss} + \lambda_3 \cdot \text{EdgeLoss}$$

## 六、重要已知限制

1. **CPU 训练限制**: 当前环境仅 CPU (PyTorch 2.11.0+cpu)，Swin3D 训练需要 GPU
2. **generate_synthetic.py CLI**: 未暴露 `target_density` 参数，需在 `main()` 中手动修改 config
3. **裂缝标签密度**: grid_res > 0.02m 时裂缝路径覆盖不足，建议 ≤ 0.01m
4. **DeformableWindowAttention3D**: 最近邻采样 (argmin) 不可微分，梯度通过 `pos_mlp` 路径回流；`offset_scale` 为明文浮点数 (不衰减)

## 七、后续阶段计划

### 阶段三 — GAN 域适应 (`models/gan/`)
- `generator.py`: PointNet++ 风格迁移生成器 (DGCNN encoder-decoder)
- `discriminator.py`: WGAN-GP 判别器
- 损失: 对抗损失 + 倒角距离 + 法向一致 + MMD (RKHS 核, 参考方法.md §2.2)

### 阶段四 — 数据加载 (`data/`)
- `dataloader.py`: RoadMCDataModule (LightningDataModule), 体素化, 数据增强
- `real/dataset.py`: 真实点云加载, JTG 标签映射

### 阶段五 — 训练与评估
- `train.py`: 三种模式 (baseline / gan_enhanced / end2end)
- `evaluate.py`: JTG 标准评估报告, 沥青/水泥分组统计

### 可用的数学方法储备
参考 `方法.md` 和 `方法2.md`:

| 方法 | 适用阶段 | 说明 |
|------|----------|------|
| 代数曲线零点集裂缝骨架 | 阶段一/二增强 | 多项式 $f(x,y)=0$ 替代 Bézier |
| Sobolev 半范数分割正则化 | 阶段二分割头 | $H^1$ 惩罚图上标签不平滑 |
| 谱方法图注意力 (Lanczos) | 阶段二注意力 | 低频子空间注意力 |
| Fredholm 积分方程注意力核 | 阶段二/三 | Green 函数力学先验 |
| MMD RKHS 核方法 | 阶段三 GAN | 分布嵌入匹配，避免模式坍塌 |
