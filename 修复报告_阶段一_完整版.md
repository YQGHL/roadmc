## P0 修复详情

### P0‑1：裂缝宽度分配逻辑修复

**文件**: `primitives.py` | **新增函数**: `_point_to_segment_distance_t()`

**问题**: `np.interp(np.linspace(0, 1, N), ...)` 按点云数组索引插值裂缝宽度，但点云是 2D 网格展平，索引位置与空间位置无关。导致裂缝标签区域随机。

**修复**: 
1. 新增 `_point_to_segment_distance_t()` — 同时返回点到线段距离和局部投影参数 `t`
2. 裂缝循环中同时追踪每个点的 `min_dist` 和 `min_t`（曲线上的全局投影参数）
3. 使用 `min_t` 替代数组索引进行宽度插值
4. 重命名深度公式变量 `t` → `depth_ratio`，避免命名混淆

### P0‑2：强度计算移至松散之后

**文件**: `generator.py`

**修复**: 将 `_compute_intensity()` 调用从步骤 5（松散应用之前）移至步骤 7（松散 NaN 过滤之后）。

### P0‑3：松散标签膨胀 + NaN 过滤

**文件**: `generator.py`

**修复**: NaN 过滤前，用 KDTree 将有效松散点（标签 11/12）的标签膨胀到周围背景点（3×grid_res 半径内），确保噪声后的 KDTree 标签传递能恢复松散标签。

### P0‑4：KDTree 标签传递距离阈值保护

**文件**: `generator.py`

**修复**: 
1. `tree.query()` 改为返回 `nn_dist` 和 `nn_idx`
2. 设置 `max_transfer_dist = distance_noise_std * 3.0`（3σ 拒绝域）
3. 超过阈值的裂缝标签（1-8）回退为背景标签 0

## P1 修复详情

### P1‑1：LiDAR 扫描线密度模拟

**文件**: `config.py` (新增 `LiDARScanConfig`)、`primitives.py` (新增 `resample_to_lidar_pattern()`)

**修复**:
1. 新增 `LiDARScanConfig` 数据类，支持 `rotate` 和 `solid_state` 两种扫描模式
2. 新增 `resample_to_lidar_pattern()` 函数：
   - 旋转式：沿 y 方向分布 scan_lines 条扫描线，每条线在 x 方向保持高密度，y 方向用 Gauss 衰减
   - 距离衰减：`P_range = 1 - α·r/r_max`
   - 入射角效应：`P_incidence = 1 - β·|x-x_center|/half_width`
   - Bernoulli 采样实现可变密度

**数学公式** (旋转式):
$$P_{\text{keep}}(x,y) = \exp\left(-\frac{d_{\text{scan}}^2}{2\sigma^2}\right) \cdot \left(1 - \alpha\frac{r}{r_{\max}}\right) \cdot \left(1 - \beta\frac{|x-x_c|}{w/2}\right)$$

### P1‑2：target_density + 体素下采样

**文件**: `config.py` (新增 `target_density` 和 `point_count_tolerance`)、`generator.py` (新增 `_voxel_downsample()`)

**修复**:
1. 新增 `target_density` (点/㎡) 配置参数，替代 `num_points`
2. 新增 `point_count_tolerance` (±20%) 允许点数自然波动
3. 实现 `_voxel_downsample()` — 2D 体素（仅 xy）质心下采样：
   - 自适应体素大小：`voxel_size = sqrt(road_area / target_count)`
   - 每个体素内：质心位置、多数投票标签、均值特征
   - 法向量重新归一化

**实测效果**: target_density=150 → 预期 5250 点，实际 5449±4 点 (±4%)，远优于 ±20% 容忍度

### P1‑3：病害标签覆盖优先级

- 病害循环中，每个基元函数应用后校验优先级，仅保留高优先级标签

## P2 修复详情

### P2‑1：Voronoi 脊线裁剪

**文件**: `primitives.py` — `add_crack()` 龟裂/块状分支

**修复**: 对每条 Voronoi 脊线段检查边界框相交性，完全在路面边界外的线段跳过，减少无效计算并避免边界虚假裂缝。

### P2‑2：KDTree 曲率计算

**文件**: `generator.py` — 新增 `_compute_kdtree_curvature()`

**问题**: 原 `_compute_grid_curvature` 通过 `z_grid.reshape(nx, ny)` 依赖网格顺序，任何点云排列变换都会导致 reshape 失败。

**修复**: 使用 KDTree 局部 PCA 曲率估计：
$$C_i = \frac{\lambda_{\min}}{\lambda_1 + \lambda_2 + \lambda_3}$$

对每个点的 k=20 近邻做协方差矩阵特征值分解，最小特征值与特征值总和之比作为局部曲率。**与点的排列顺序无关**。

**与 `方法.md` 和 `方法2.md` 的关联**: 方法文档中提出的"代数曲面的高斯曲率符号分布判别破损类型"思路与此一致——PCA 曲率正比于代数曲面在该点的高斯曲率绝对值。裂缝区域的曲率震荡剧烈（高 λ_min/Σλ），这正是 P2-3 的边缘检测阈值基础。

### P2‑3：边缘混合可配置 + 仅高曲率点混合

**文件**: `primitives.py` — `simulate_lidar_noise()`

**修复**:
1. 新增 `enable_edge_mixing` 参数（默认 True，可在 `LidarNoiseConfig` 中关闭）
2. 新增 `curvature` 和 `curvature_threshold` 参数
3. 当提供曲率时，仅对高曲率（>0.5σ）边缘点做混合——避免平滑平坦区域的病害信号
4. **批量化 KDTree 查询**（可改进点#5）：从逐点查询改为单次批量查询 `tree.query(..., k=6)`

### P2‑4：软标签 — 裂缝边界距离特征

**文件**: `generator.py` — `generate_scene()` 步骤 10

**修复**: 为每个点计算到最近裂缝边界的欧式距离，转换为裂缝概率软标签：
1. 找出裂缝标签点 (1-8) 和背景点
2. 双向 KDTree 查询：背景→裂缝距离 + 裂缝→背景距离
3. 将裂缝边界距离 `crack_boundary_dist` 作为第三个特征通道加入 feats
4. 输出 feats 从 `(N, 2)` 扩展为 `(N, 3)`: `[intensity, curvature, crack_boundary_dist]`

**软标签语义**: 
- `crack_boundary_dist ≈ 0`: 确定性的裂缝区域或远离裂缝的区域（取决于 label 上下文）
- `crack_boundary_dist > 0`: 裂缝边界过渡带，距离越大意味着越接近裂缝边界
        
下游训练中，可将 `crack_boundary_dist` 转换为概率值 `exp(-d²/2σ²)` 用于 label smoothing 或边界加权损失。

**实测**: 有裂缝的场景 crack_boundary_dist 范围 [0.005, 0.647]，无裂缝场景为 [0, 0]。

## 可改进点修复详情

### #1：`add_raveling` 增加 `remove_nan` 参数

**文件**: `primitives.py`

新增 `remove_nan: bool = False` 参数，为 `True` 时在返回前自动过滤 NaN 点并裁剪标签，减少调用方负担。

### #5：LiDAR 边缘混合性能优化 (批量化)

**文件**: `primitives.py` — `simulate_lidar_noise()`

从 O(n·k·log n) 的逐点 KDTree 查询改为 O(n·log n) 的单次批量查询：
```python
# 修复前 (for loop):
for idx in mixed_idx:
    dists, neighbors = tree.query(pts_noisy[idx], k=6)

# 修复后 (batch):
_, all_neighbors = tree.query(pts_noisy[mixed_idx], k=6)
neighbors_mean = np.mean(pts_noisy[all_neighbors[:, 1:]], axis=1)
```

```

10. 体素下采样 (P1-2) 或 num_points 重采样
11. 坐标归一化
12. 特征拼接 → 输出 {points, labels, feats, normals, pavement_type}
```
