## P0 修复详情

### P0‑1：裂缝宽度分配逻辑修复

**文件**: `primitives.py`

**问题**: `add_crack()` 函数中纵向/横向裂缝的宽度分配使用 `np.interp(np.linspace(0, 1, N), ...)` 按点云数组索引插值，但点云是 2D 网格展平的结果，索引位置与裂缝曲线上的空间位置无关。导致裂缝宽度区域与真实裂缝路径不匹配。

**修复方案**:
1. 新增辅助函数 `_point_to_segment_distance_t()`，在计算点到线段距离的同时返回局部投影参数 `t_local`
2. 在裂缝曲线遍历循环中，同时跟踪每个点的最小距离和对应的曲线参数 `min_t`
3. 使用 `min_t`（空间投影参数）替代数组索引进行宽度插值

```python
# 修复前（错误）：
half_width = np.interp(
    np.linspace(0, 1, N),          # ← N 个点的索引，非空间位置
    np.linspace(0, 1, num_curve_samples),
    lognorm_sample,
) / 2.0

# 修复后（正确）：
half_width = np.interp(
    min_t,                          # ← 基于空间投影参数的插值
    np.linspace(0, 1, num_curve_samples),
    lognorm_sample,
) / 2.0
```

**同时修复**: 深度公式中的变量名 `t` 重命名为 `depth_ratio`，避免与投影参数 `min_t` 混淆。

### P0‑2：强度计算移至松散标签应用之后

**文件**: `generator.py`

**问题**: `_compute_intensity()` 在步骤 5 调用，此时 `labels` 尚未包含松散标签（11/12）。松散在步骤 6 应用后，松散区域的强度仍保持背景值，不反映真实反射率变化。

**修复方案**: 将 `_compute_intensity()` 调用从步骤 5 移至步骤 7（松散 NaN 过滤之后、LiDAR 噪声之前）。

```
# 修复前：
4. 网格曲率 → 5. 强度计算 → 6. 松散 → 7. NaN过滤 → 8. LiDAR噪声

# 修复后：
4. 网格曲率 → 5. 松散 → 6. NaN过滤+标签膨胀 → 7. 强度计算 → 8. LiDAR噪声
```

### P0‑3：松散标签膨胀 + NaN 过滤修复

**文件**: `generator.py`

**问题**: `add_raveling()` 将被移除的点设为 `NaN`（整行），后续 `valid_mask` 过滤直接丢弃这些点及其标签。LiDAR 噪声后落在原松散区域的点，通过 KDTree 最近邻找到的是非松散邻居，标签错误。

**修复方案**: 在 NaN 过滤之前，执行标签膨胀：
1. 构建有效点 KDTree
2. 查询背景点到最近松散标签点的距离
3. 在 3×`grid_res` 半径内的背景点，标记为对应松散标签

这确保了松散区域的 "缓冲带"，使得噪声仿真后的近邻传递能正确恢复松散标签。

### P0‑4：KDTree 标签传递距离阈值保护

**文件**: `generator.py`

**问题**: `tree.query(noisy_points)` 无条件传递最近邻标签。裂缝边缘噪声点移动到健康区域后仍携带裂缝标签（标签溢出），反之健康点移入裂缝区域则获得背景标签（标签侵蚀）。

**修复方案**:
1. `tree.query()` 改为返回距离 `nn_dist` 和索引 `nn_idx`
2. 设置距离阈值 `max_transfer_dist = distance_noise_std * 3.0`（3σ 拒绝域）
3. 超过阈值的裂缝标签（1-8）回退为背景标签 0

```python
max_transfer_dist = self.config.lidar_noise.distance_noise_std * 3.0
uncertain_mask = nn_dist > max_transfer_dist
for i in range(len(labels)):
    if uncertain_mask[i] and labels[i] in crack_labels:
        labels[i] = 0  # 不确定的裂缝标签降为背景
```

**原因**: 需要替换 `num_points` 配置参数为 `target_density`（点/㎡），并实现体素下采样/FPS 替代随机重采样。影响 `GeneratorConfig` 和恢复管线。

**建议方案**: 用 `target_density` 替代 `num_points`，使用 FPS 或体素质心下采样，允许最终点数 ±20% 波动。
